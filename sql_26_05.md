Разработаем архитектурную задачу по теме **"Платформа для бронирования мероприятий"** 

---

### Задача: Система бронирования мероприятий

**Описание**:  
Вы разрабатываете базу данных для платформы, где пользователи могут бронировать билеты на мероприятия (концерты, выставки, мастер-классы). Каждое мероприятие проводится в определенном месте, имеет ограниченное количество мест и дату проведения. Пользователи могут бронировать одно или несколько мест на мероприятие. Система должна хранить информацию о пользователях, мероприятиях, местах проведения и бронированиях.

**Требования к базе данных**:
1. Создайте таблицы с правильными типами данных, первичными и внешними ключами.
2. Добавьте ограничения для обеспечения целостности данных (например, уникальность, обязательные поля, проверка диапазонов).
3. Вставьте тестовые данные (не менее 5 пользователей, 3 места проведения, 5 мероприятий и 10 бронирований).
4. Напишите SQL-запросы для проверки корректности работы системы.

**Сущности и их атрибуты**:
1. **Пользователи** (Users):
   - ID пользователя (уникальный идентификатор).
   - Имя (строка, обязательно).
   - Email (уникальный, строка).
   - Дата регистрации (дата и время).
2. **Места проведения** (Venues):
   - ID места (уникальный идентификатор).
   - Название места (строка, обязательно).
   - Адрес (строка, обязательно).
   - Вместимость (целое число, больше 0).
3. **Мероприятия** (Events):
   - ID мероприятия (уникальный идентификатор).
   - Название мероприятия (строка, обязательно).
   - Дата и время проведения (дата и время, обязательно).
   - ID места проведения (связь с таблицей Venues).
   - Общее количество мест (целое число, должно быть меньше или равно вместимости места).
   - Цена билета (десятичное число, больше или равно 0).
4. **Бронирования** (Bookings):
   - ID бронирования (уникальный идентификатор).
   - ID пользователя (связь с таблицей Users).
   - ID мероприятия (связь с таблицей Events).
   - Количество забронированных мест (целое число, больше 0).
   - Дата бронирования (дата и время, обязательно).
   - Статус бронирования (например, "confirmed", "canceled").

**Ограничения**:
- Email пользователя должен быть уникальным.
- Вместимость места проведения и количество мест на мероприятии должны быть положительными.
- Дата проведения мероприятия должна быть в будущем (на момент создания).
- Статус бронирования должен быть из ограниченного набора значений.

**Задания**:
1. Создать SQL-скрипт для генерации всех таблиц с указанными полями, типами данных, первичными и внешними ключами, а также ограничениями.
2. Вставить тестовые данные:
   - 5 пользователей.
   - 3 места проведения.
   - 5 мероприятий.
   - 10 бронирований (включая случаи, где одно мероприятие бронируют несколько пользователей).
3. Написать SQL-запросы для проверки:
   - Список всех мероприятий с указанием места проведения и оставшихся мест.
   - Список бронирований конкретного пользователя (по email).
   - Общая выручка по каждому мероприятию (цена билета × количество забронированных мест).
   - Мероприятия, у которых не осталось свободных мест.
