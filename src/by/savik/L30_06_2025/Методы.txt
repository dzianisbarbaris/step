
### Основные HTTP-методы в AJAX
AJAX позволяет отправлять запросы к серверу с использованием различных HTTP-методов, каждый из которых имеет своё назначение:
- **GET**: Получение данных (например, списка ресурсов).
- **POST**: Создание нового ресурса.
- **PUT**: Полное обновление существующего ресурса.
- **DELETE**: Удаление ресурса.
- **PATCH**: Частичное обновление ресурса.

Я использую `fetch` с `async/await`, так как это современный и читаемый подход. Все примеры будут работать с ресурсом `/posts` на `https://jsonplaceholder.typicode.com`, но вы можете адаптировать их под любой API.

---

### Примеры по каждому методу

#### 1. GET: Получение данных
**Объяснение**: GET-запрос запрашивает данные с сервера, например, список постов или один пост по ID. Не содержит тела запроса.

**Пример**:
```javascript
async function getPost(id) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
    if (!response.ok) throw new Error('Ошибка сети');
    const post = await response.json();
    console.log('GET:', post);
  } catch (error) {
    console.error('Ошибка:', error);
  }
}
getPost(1);
```

**Как объяснить**:
- GET — самый простой метод, используется для чтения данных.
- Покажите JSON-ответ в консоли (содержит `id`, `title`, `body`, `userId`).
- Упомяните, что параметры можно добавлять в URL (например, `?_limit=5`).

#### 2. POST: Создание ресурса
**Объяснение**: POST-запрос отправляет данные на сервер для создания нового ресурса. Требует тело запроса (обычно JSON).

**Пример**:
```javascript
async function createPost() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ title: 'Новый пост', body: 'Содержимое', userId: 1 })
    });
    if (!response.ok) throw new Error('Ошибка сети');
    const newPost = await response.json();
    console.log('POST:', newPost);
  } catch (error) {
    console.error('Ошибка:', error);
  }
}
createPost();
```

**Как объяснить**:
- POST создаёт новый ресурс, сервер возвращает созданный объект (с новым `id`).
- Объясните `headers` и `JSON.stringify` для отправки JSON.
- Покажите, что JSONPlaceholder имитирует создание, возвращая объект с `id`.

#### 3. PUT: Полное обновление ресурса
**Объяснение**: PUT-запрос заменяет существующий ресурс полностью. Требует указания ID и полного набора данных.

**Пример**:
```javascript
async function updatePost(id) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ id, title: 'Обновлённый пост', body: 'Новое содержимое', userId: 1 })
    });
    if (!response.ok) throw new Error('Ошибка сети');
    const updatedPost = await response.json();
    console.log('PUT:', updatedPost);
  } catch (error) {
    console.error('Ошибка:', error);
  }
}
updatePost(1);
```

**Как объяснить**:
- PUT обновляет весь ресурс, заменяя старые данные.
- Сравните с POST: PUT требует ID, POST создаёт новый ресурс.
- Покажите, что сервер возвращает обновлённый объект.

#### 4. DELETE: Удаление ресурса
**Объяснение**: DELETE-запрос удаляет ресурс по указанному ID. Обычно не содержит тела.

**Пример**:
```javascript
async function deletePost(id) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
      method: 'DELETE'
    });
    if (!response.ok) throw new Error('Ошибка сети');
    console.log('DELETE: Успешно удалено');
  } catch (error) {
    console.error('Ошибка:', error);
  }
}
deletePost(1);
```

**Как объяснить**:
- DELETE удаляет ресурс, сервер обычно возвращает пустой ответ (статус 200).
- Покажите, что JSONPlaceholder имитирует удаление (реальный ресурс не удаляется).
- Упомяните, что тело запроса обычно не нужно.

#### 5. PATCH: Частичное обновление ресурса
**Объяснение**: PATCH обновляет только указанные поля ресурса, не трогая остальные.

**Пример**:
```javascript
async function patchPost(id) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ title: 'Частично обновлённый пост' })
    });
    if (!response.ok) throw new Error('Ошибка сети');
    const patchedPost = await response.json();
    console.log('PATCH:', patchedPost);
  } catch (error) {
    console.error('Ошибка:', error);
  }
}
patchPost(1);
```

**Как объяснить**:
- PATCH меняет только указанные поля (например, только `title`).
- Сравните с PUT: PATCH более экономичен, так как отправляет меньше данных.
- Покажите, что сервер возвращает полный объект с обновлённым полем.

### Интерактивный демо-код
Теперь я создам демо-страницу, которая позволит студентам протестировать все методы (GET, POST, PUT, DELETE, PATCH) на ресурсе `/posts`. Страница будет включать поля ввода для ID и данных, кнопки для каждого метода и область вывода результатов.

#### demo.html
```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Демо AJAX: Все методы</title>
  <style>
    body { font-family: Arial, sans-serif; margin: 20px; }
    .container { max-width: 600px; margin: auto; }
    button { padding: 8px; margin: 5px; cursor: pointer; }
    input { padding: 5px; margin: 5px; width: 200px; }
    #output { border: 1px solid #ccc; padding: 10px; margin-top: 10px; min-height: 100px; }
    label { display: inline-block; width: 100px; }
  </style>
</head>
<body>
  <div class="container">
    <h1>Демо AJAX: Все методы</h1>
    
    <div>
      <label for="postId">ID поста (1-100):</label>
      <input id="postId" type="number" placeholder="Введите ID">
    </div>
    <div>
      <label for="postTitle">Заголовок:</label>
      <input id="postTitle" placeholder="Введите заголовок">
    </div>
    <div>
      <label for="postBody">Текст:</label>
      <input id="postBody" placeholder="Введите текст">
    </div>
    
    <button id="getBtn">GET: Получить пост</button>
    <button id="postBtn">POST: Создать пост</button>
    <button id="putBtn">PUT: Обновить пост</button>
    <button id="patchBtn">PATCH: Частично обновить</button>
    <button id="deleteBtn">DELETE: Удалить пост</button>
    
    <h3>Результат:</h3>
    <div id="output">Здесь будет результат</div>
  </div>

  <script src="demo.js"></script>
</body>
</html>
```

#### demo.js
```javascript
document.addEventListener('DOMContentLoaded', () => {
  const postIdInput = document.getElementById('postId');
  const postTitleInput = document.getElementById('postTitle');
  const postBodyInput = document.getElementById('postBody');
  const getBtn = document.getElementById('getBtn');
  const postBtn = document.getElementById('postBtn');
  const putBtn = document.getElementById('putBtn');
  const patchBtn = document.getElementById('patchBtn');
  const deleteBtn = document.getElementById('deleteBtn');
  const output = document.getElementById('output');

  // Валидация ID
  const validateId = (id) => {
    id = id.trim();
    if (!id || id < 1 || id > 100) {
      output.textContent = 'Введите ID от 1 до 100';
      return false;
    }
    return id;
  };

  // GET: Получение поста
  getBtn.addEventListener('click', async () => {
    const id = validateId(postIdInput.value);
    if (!id) return;

    try {
      output.textContent = 'Загрузка...';
      const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
      if (!response.ok) throw new Error('Ошибка сети');
      const post = await response.json();
      output.textContent = `GET:\nID: ${post.id}\nЗаголовок: ${post.title}\nТекст: ${post.body}`;
    } catch (error) {
      output.textContent = `Ошибка: ${error.message}`;
    }
  });

  // POST: Создание поста
  postBtn.addEventListener('click', async () => {
    const title = postTitleInput.value.trim();
    const body = postBodyInput.value.trim();
    if (!title || !body) {
      output.textContent = 'Заполните заголовок и текст';
      return;
    }

    try {
      output.textContent = 'Отправка...';
      const response = await fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ title, body, userId: 1 })
      });
      if (!response.ok) throw new Error('Ошибка сети');
      const newPost = await response.json();
      output.textContent = `POST:\nID: ${newPost.id}\nЗаголовок: ${newPost.title}\nТекст: ${newPost.body}`;
      postTitleInput.value = '';
      postBodyInput.value = '';
    } catch (error) {
      output.textContent = `Ошибка: ${error.message}`;
    }
  });

  // PUT: Полное обновление поста
  putBtn.addEventListener('click', async () => {
    const id = validateId(postIdInput.value);
    if (!id) return;
    const title = postTitleInput.value.trim();
    const body = postBodyInput.value.trim();
    if (!title || !body) {
      output.textContent = 'Заполните заголовок и текст';
      return;
    }

    try {
      output.textContent = 'Отправка...';
      const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ id, title, body, userId: 1 })
      });
      if (!response.ok) throw new Error('Ошибка сети');
      const updatedPost = await response.json();
      output.textContent = `PUT:\nID: ${updatedPost.id}\nЗаголовок: ${updatedPost.title}\nТекст: ${updatedPost.body}`;
    } catch (error) {
      output.textContent = `Ошибка: ${error.message}`;
    }
  });

  // PATCH: Частичное обновление поста
  patchBtn.addEventListener('click', async () => {
    const id = validateId(postIdInput.value);
    if (!id) return;
    const title = postTitleInput.value.trim();
    if (!title) {
      output.textContent = 'Введите заголовок';
      return;
    }

    try {
      output.textContent = 'Отправка...';
      const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ title })
      });
      if (!response.ok) throw new Error('Ошибка сети');
      const patchedPost = await response.json();
      output.textContent = `PATCH:\nID: ${patchedPost.id}\nЗаголовок: ${patchedPost.title}\nТекст: ${patchedPost.body}`;
    } catch (error) {
      output.textContent = `Ошибка: ${error.message}`;
    }
  });

  // DELETE: Удаление поста
  deleteBtn.addEventListener('click', async () => {
    const id = validateId(postIdInput.value);
    if (!id) return;

    try {
      output.textContent = 'Удаление...';
      const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
        method: 'DELETE'
      });
      if (!response.ok) throw new Error('Ошибка сети');
      output.textContent = `DELETE: Пост ${id} успешно удалён`;
    } catch (error) {
      output.textContent = `Ошибка: ${error.message}`;
    }
  });
});
```

### Как использовать на занятии

1. **Объяснение методов**:
   - Начните с объяснения каждого метода, используя примеры выше:
     - **GET**: Для получения данных, не меняет сервер.
     - **POST**: Создаёт новый ресурс, отправляет данные.
     - **PUT**: Заменяет весь ресурс, требует полный объект.
     - **PATCH**: Обновляет только указанные поля.
     - **DELETE**: Удаляет ресурс, обычно без тела.
   - Для каждого метода:
     - Покажите код в консоли или отдельном файле.
     - Объясните структуру запроса (`method`, `headers`, `body`).
     - Продемонстрируйте JSON-ответ в консоли (кроме DELETE).

2. **Демонстрация демо-кода**:
   - Сохраните `demo.html` и `demo.js` в одной папке.
   - Откройте `demo.html` через локальный сервер (например, `Live Server` в VS Code), так как AJAX требует HTTP/HTTPS.
   - Протестируйте каждый метод:
     - **GET**: Введите ID (например, 1), нажмите "Получить пост" — результат в `#output`.
     - **POST**: Введите заголовок и текст, нажмите "Создать пост" — увидите новый пост с `id`.
     - **PUT**: Введите ID, заголовок и текст, нажмите "Обновить пост" — обновлённый пост.
     - **PATCH**: Введите ID и заголовок, нажмите "Частично обновить" — только заголовок обновится.
     - **DELETE**: Введите ID, нажмите "Удалить пост" — сообщение об удалении.
     - Введите некорректные данные (например, пустой заголовок или ID > 100), чтобы показать валидацию.
   - Откройте DevTools → Network, чтобы показать HTTP-запросы для каждого метода.

3. **Пошаговое объяснение кода**:
   - **DOMContentLoaded**: Код ждёт загрузки DOM.
   - **Поиск элементов**: `getElementById` для `<input>` и `<button>`.
   - **Валидация**: Функция `validateId` проверяет ID.
   - **Запросы**: Разберите структуру `fetch` для каждого метода, подчеркнув различия (`method`, наличие `body`).
   - **Ошибки**: Покажите `try/catch` и проверки ввода.
   - **Очистка полей**: Укажите на `postTitleInput.value = ''` после POST.

4. **Задания для студентов**:
   - Добавить кнопку для получения списка постов (`GET /posts?_limit=5`) и отображения их в `<ul>`.
   - Реализовать валидацию длины заголовка (минимум 5 символов) для POST и PUT.
   - Добавить индикатор загрузки (например, `<div>` с `display: none/block`) вместо `textContent = 'Загрузка...'`.
   - Переписать один метод (например, GET) с использованием `XMLHttpRequest`.
   - Добавить кнопку для очистки полей ввода и `#output`.
